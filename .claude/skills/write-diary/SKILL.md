# 日記を書く

セッション中の開発内容、反省点、学びを記録する。

## 手順

1. 現在の日付を取得する

```bash
pnpm get-time
```

出力例: `2025-01-13 14:30 JST`

2. `/docs/diary/YYYYMMDD.md` を作成または追記する

3. `/docs/diary/CLAUDE.md` のガイドラインに従って記述する

4. **CLAUDE.mdへの反映を検討する**

日記に書いた反省点・学びの中で、以下に該当するものは `CLAUDE.md` に追記を検討する:

- 繰り返し発生しそうな問題 → 「開発ワークフローのルール」に追加
- 具体的なコードの落とし穴 → 「既知の落とし穴」テーブルに追加
- 新しいスキルの必要性 → `.claude/skills/` に新規スキル作成

同じ反省を何度も日記に書いている場合、それはルール化されていないということ。

## 注意

- 日記を書くことが目的化しないようにする
- 書いた反省が行動に反映されているか、定期的に振り返る
