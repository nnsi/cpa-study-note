# シードデータフォーマット

## 科目・論点CSV

### ファイル配置

```
csv/
├── 財務会計論_出題項目.csv  ✓ 抽出済み
├── 管理会計論_出題項目.csv  ✓ 抽出済み
├── 監査論_出題項目.csv      ✓ 抽出済み
├── 企業法_出題項目.csv      ✓ 抽出済み
├── 租税法_出題項目.csv      ✓ 抽出済み
├── 経営学_出題項目.csv      ✓ 抽出済み
├── 経済学_出題項目.csv      ✓ 抽出済み
└── 民法_出題項目.csv        ✓ 抽出済み
```

### CSVフォーマット

```csv
科目名,大項目,中項目,小項目
財務会計論,財務会計の意義と機能,財務会計の意義,
財務会計論,財務会計の意義と機能,財務会計の機能,情報提供機能
財務会計論,財務会計の意義と機能,財務会計の機能,利害調整機能
財務会計論,財務会計の基礎概念,会計公準,企業実体の公準
```

### カラム定義

| カラム | 必須 | 説明 |
|--------|------|------|
| 科目名 | ✓ | 科目名（財務会計論、監査論等） |
| 大項目 | ✓ | 第1階層（depth=1） |
| 中項目 | ✓ | 第2階層（depth=2） |
| 小項目 | | 第3階層（depth=3）、空の場合は中項目が末端 |

### 階層構造

```
財務会計論（科目）
├── 財務会計の意義と機能（大項目）
│   ├── 財務会計の意義（中項目）← 小項目なし = 末端
│   └── 財務会計の機能（中項目）
│       ├── 情報提供機能（小項目）
│       └── 利害調整機能（小項目）
└── 財務会計の基礎概念（大項目）
    └── 会計公準（中項目）
        ├── 企業実体の公準（小項目）
        ├── 会計期間の公準（小項目）
        └── 貨幣的測定の公準（小項目）
```

### インポート処理

1. **科目**: 科目名からユニークな科目を抽出、IDを自動生成
2. **大項目**: 科目×大項目でユニーク、depth=1
3. **中項目**: 科目×大項目×中項目でユニーク、depth=2、親=大項目
4. **小項目**: 空でなければ登録、depth=3、親=中項目

### ID自動生成ルール

- 科目ID: 科目名のスラッグ化（`財務会計論` → `financial`）またはUUID
- 論点ID: UUID または `{科目ID}-{連番}`

### 注意事項

- CSVはUTF-8（BOMなし推奨、BOM付きも対応）
- 改行はLF or CRLF
- 小項目が空の行 = 中項目が末端論点
- 同じ「科目×大項目×中項目×小項目」の重複行は無視
